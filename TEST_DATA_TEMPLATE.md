## 一、典型测试用例（推荐优先执行）

### Case 1：加仓已持有标的（仓位超限）

**输入**
```
理想汽车下周有新车发布会，我想在发布会前加仓，合适吗？
```

**预期流程结果**

| 阶段 | 预期输出 |
|------|---------|
| 意图解析 | asset=理想汽车, action_type=加仓判断, trigger=发布会, confidence≥0.8 |
| 持仓匹配 | 命中「理想汽车 (LI)」，仓位约 22.2% |
| 规则校验 | violation=True（22.2% > 单标上限 15%），显示「超限 ⛔」 |
| 信号层 | position_signal=偏高, event_signal=不确定性高, fundamental=正面（有内置 mock 观点） |
| AI 结论 | 倾向 HOLD 或 SELL（因仓位已超限） |

---

### Case 2：减仓未持有标的（规则违规）

**输入**
```
我想卖出贵州茅台，你觉得现在价格合适吗？
```

**预期流程结果**

| 阶段 | 预期输出 |
|------|---------|
| 意图解析 | asset=贵州茅台, action_type=卖出判断 |
| 持仓匹配 | target_position=None（持仓中无此标的） |
| 规则校验 | violation=True，显示「未持有该标的，无法减仓」 |
| 信号层 | position_signal=偏低（ratio=0），fundamental=N/A |
| AI 结论 | 倾向 HOLD 或提示无法减仓 |

---

### Case 3：新建仓买入（正常流程）

**输入**
```
我想买入苹果，当前时机合适吗？
```

**预期流程结果**

| 阶段 | 预期输出 |
|------|---------|
| 意图解析 | asset=苹果, action_type=买入判断 |
| 持仓匹配 | 命中「苹果 (AAPL)」（若已持有） |
| 规则校验 | 视当前苹果仓位决定 violation / warning |
| 信号层 | 依持仓比率决定 position_signal |
| AI 结论 | BUY / HOLD 视综合信号而定 |

## 二、边界用例（重点验证稳健性）

| 编号 | 输入 | 预期行为 |
|------|------|---------|
| E1 | `（空输入，直接点分析）` | 按钮 disabled，无法提交 |
| E2 | `买啥好` | 置信度 < 0.6，返回澄清问题，流程中断 |
| E3 | `   ` （纯空格） | 同 E1 |
| E4 | `我想了解一下市场` | 无明确标的，置信度低，触发澄清 |
| E5 | `Meta仓位太重，要减吗？` | 依 Meta 实际持仓判断，若无持有则 violation |
| E6 | `我要加仓英伟达英伟达英伟达！！！！` | 应正常解析，不因感叹号崩溃 |
| E7 | 中英混合：`我要加 AAPL，合适吗` | 意图解析应能识别 AAPL → 苹果 |
| E8 | 超长输入（500+ 字的分析文本） | 系统应正常处理，不超时崩溃 |

## 三、信号层验证速查表

| 仓位比率（当前/上限） | position_signal | 说明 |
|-------------------|-----------------|----|
| > 80%（如 22%/15%=148%） | 偏高 | 规则同时 violation |
| 40%~80%（如 12%/25%=48%） | 合理 | 正常区间 |
| < 40%（如 5%/25%=20%） | 偏低 | 可加仓空间大 |
| 0%（未持有） | 偏低 | 新建仓场景 |

| 是否有 trigger | event_signal.uncertainty |
|--------------|--------------------------|
| 有（发布会、财报等） | 高 |
| 无 | 低 |

| 投研观点内容 | fundamental_signal |
|------------|-------------------|
| 含"看好"/"增长"/"超预期"等正面词 | 正面 |
| 含"风险"/"下滑"/"承压"等负面词 | 负面 |
| 无明显倾向 | 中性 |
| 无观点 / 仅"暂无" | N/A |

## 五、API Key 测试注意事项

- **不设置 API Key**：页面在「🔑」警告后调用 `st.stop()`，分析按钮不可用
- **设置无效 Key**：意图解析阶段抛 `AuthenticationError`，流程 ABORTED，显示错误
- **API 超时**：LLM Engine 有超时捕获，降级返回 HOLD，展示"系统繁忙"提示
- **正常 Key**：全流程约 3~10 秒完成
