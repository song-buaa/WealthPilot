# WealthPilot 投资决策页 UI结构规范

## 🧩 一、整体布局（Layout System）

### 1. 页面结构

┌──────────────────────────────────────────────┐  
│ Sidebar（固定）                              │  
├──────────────────────────────────────────────┤  
│                Main Content                  │  
│  ┌───────────────┬────────────────────────┐  │  
│  │ Left: Chat    │ Right: Explain Panel   │  │  
│  └───────────────┴────────────────────────┘  │  
└──────────────────────────────────────────────┘

---

## 2. 左右布局比例（必须固定）

* 默认：**50% / 50%**
* 最小宽度：
  * Chat ≥ 520px
  * Panel ≥ 420px

禁止：  
❌ 7:3  
❌ 自动塌缩  
❌ 内容撑开布局

---

## 3. 高度规则（非常关键）

整个页面高度：

height = 100vh（不允许页面整体滚动）

左右两侧：

| 区域            | 是否独立滚动 |
| ------------- | ------ |
| Chat消息区       | ✅      |
| Explain Panel | ✅      |
| 页面整体          | ❌      |

## 实现约束（必须执行）

必须使用 CSS Flex 布局实现：

- Chat Container：
  display: flex
  flex-direction: column
  height: 100%

- Message List：
  flex: 1
  overflow-y: auto

- Input Area：
  position: sticky 或 flex 固定底部

禁止：
❌ 通过固定 height 数值模拟布局（如 height=480）
❌ 通过 Streamlit container 高度 hack 实现

---

# 💬 二、左侧 Chat 区（核心体验区）

## 1. 结构（必须严格）

Chat Container  
│  
├── Header（标题：对话）  
├── Message List（可滚动）  
└── Input Area（固定底部）

---

## 2. 布局细节

### Header

* 使用 H2
* 左对齐
* 与内容间距：16px

---

### Message List（重点）

* `flex: 1`
* `overflow-y: auto`
* padding：16px

---

### Input Area（关键）

必须满足：

* 固定在底部
* 永远可见
* 不随消息滚动

结构：

┌────────────────────────────┐  
│ [输入框………………]   [发送]   │  
└────────────────────────────┘

样式：

* 上边框（弱分割线）
* padding：12px–16px
* 背景：白色

---

## 3. 单条消息结构（重点）

### 用户消息

[icon] 用户问题

* 字号：Body
* 行高：1.5

---

### AI消息（必须分层）

AI回答（自然语言总结）  

（空行）  

详细解释段落  

（空行）  

【操作建议】  
• xxx  
• xxx  

（空行）  

【风险提示】  
• xxx

---

## 4. Chat排版规则（必须执行）

| 项目        | 规则     |
| --------- | ------ |
| 段落间距      | ≥ 10px |
| 模块间距      | ≥ 14px |
| 标题（建议/风险） | Title  |
| 正文        | Body   |
| 说明        | Desc   |

禁止：  
❌ 一整块文本  
❌ 无分段  
❌ 所有文字同样式

## 5. AI回答强制格式（必须执行）

AI回答必须拆成以下结构：

1. 第一段：结论（1-2句）
2. 第二段：核心原因（1-2段）
3. 第三段：操作建议（bullet list）
4. 第四段：风险提示（bullet list）

每一段之间必须有空行

禁止：
❌ 单段超过5行文本
❌ 无结构输出

---

# 🧠 三、右侧 Explain Panel（决策黑箱）

## 1. 结构（必须分块）

Explain Panel Container（独立滚动）  
│  
├── Header（决策链路）  
├── 模块1：意图解析  
├── 模块2：持仓数据（可折叠）  
├── 模块3：规则校验  
├── 模块4：信号层  
└── 模块5：推理过程（可折叠）

---

## 2. 容器规则

* 固定高度（≈ 页面高度 - padding）
* `overflow-y: auto`
* 有边框（非常重要）

border: 1px solid #E5E7EB  
border-radius: 8px

---

## 3. 模块设计（核心）

每个模块必须：

┌────────────────────┐  
│ 模块标题           │  
│ 内容内容内容       │  
└────────────────────┘

样式：

* padding：12px
* 模块间距：12–16px
* 背景：白 or 浅灰（交替可选）

---

## 4. 字段展示规范（非常关键）

### 正确结构：

LABEL（小字）  
value（正文）

例：

标的  
理想汽车  

操作类型  
减仓判断

---

### 字体绑定（必须）

| 类型      | Token |
| ------- | ----- |
| 模块标题    | Title |
| 字段Label | Label |
| 字段值     | Body  |
| 辅助信息    | Meta  |

---

## 5. 折叠策略（重要）

默认：

| 模块   | 状态  |
| ---- | --- |
| 意图解析 | 展开  |
| 规则校验 | 展开  |
| 信号层  | 展开  |
| 持仓数据 | 折叠  |
| 推理过程 | 折叠  |

## 6. 模块视觉规则（必须执行）

每个模块必须具备：

1. 独立 container（不是连续文本）
2. 上下间距 ≥ 12px
3. 内部 padding ≥ 12px
4. 有视觉边界（以下二选一）：
   - 浅边框
   - 浅背景（#F9FAFB）

模块之间必须有明显分隔，不允许连成一整块

# 📏 五、间距系统（Spacing）

统一间距规则：

| 类型         | 数值   |
| ---------- | ---- |
| 页面 padding | 24px |
| 模块间距       | 16px |
| 内容块间距      | 12px |
| 段落间距       | 10px |

---

# ⚠️ 六、必须避免的错误（你现在版本有）

### ❌ 错误1：Chat像报告

→ 必须拆段 + 分组

### ❌ 错误2：右侧像长表单

→ 必须模块化 + 折叠

### ❌ 错误3：字体层级混乱

→ 必须语义绑定

### ❌ 错误4：页面整体滚动

→ 必须左右独立滚动

---

# ✅ 七、验收标准

---

### 1️⃣ 结构

* 输入框是否永远贴底
* 左右是否独立滚动

---

### 2️⃣ Chat

* 是否有明显分段
* 是否有“建议 / 风险”结构

---

### 3️⃣ Panel

* 是否分模块
* 是否有Label / Value区分

---

### 4️⃣ 字体

* 是否一眼能看出层级
* 是否没有大小混乱
